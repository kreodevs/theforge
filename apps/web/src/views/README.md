# views

Vistas de la aplicación web.

| Componente           | Uso                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| -------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **WorkshopView.tsx** | Vista principal de entrevista: grid de 3 columnas (Tailwind). Col A: ChatContainer (useInterview). Col B: tabs (Paso 0, MDD, Guía UX/UI, Blueprint, API, Flujos, Infra) con contenido en tiempo real (react-markdown o textarea). Col C: Semáforo + costos. Estado compartido con Zustand. **Paso 0:** "Generar Benchmark" llama a `POST /ai-analysis/stream` (agentes LangGraph): en el chat se muestra el progreso de cada agente (Scout, Auditor, Critic, Synthesis) y al terminar se persiste el documento markdown en `dbgaContent`. Cuando hay benchmark, el botón **"Generar MDD con agentes"** llama a `POST /ai-analysis/mdd/stream` (pipeline Clarifier → … → Auditor); al terminar se cambia a la pestaña MDD y se muestra un banner: "Revisa el MDD en esta pestaña y refina con el chat si algo no cuadra." "Generar Deep Research" llama a `phase0-deep-research` y muestra `phase0SummaryContent`. La pestaña **Guía UX/UI** usa conversación con IA y persiste en `uxUiGuideContent`. |

Se entra desde la lista de proyectos en App; se pasa `projectId`, `projectName` y `onBack`.
