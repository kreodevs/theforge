# store

Estado global con Zustand.

| Archivo              | Uso                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| -------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **workshopStore.ts** | Exporta **`sessionMessageBody`** para `POST /sessions/:id/messages` con `stageId` opcional. Store del Workshop: project (incluye **`stages[]`**, **`complexityPending`** HITL), **`activeStageId`** (foco MDD: semáforo/MDD mostrados vía `workshopFlatFromStage`), session, mddContent, loading, synced, error, **agentProgress**, **evaluatorCritique** (texto del evaluador en el evento SSE `done` del orquestador; no Relic). **`confirmComplexityProposal` / `dismissComplexityProposal`** para el banner de complejidad. Tras stream DBGA (`/ai-analysis/stream`), si el `done` incluye `complexityProposal`, se actualiza el proyecto en memoria antes de `fetchProject`. Al entrar al flujo MDD/Manager o al terminar/error en ese stream se pone `null` para no dejar texto obsoleto al cambiar de flujo. **`clearChat`** borra también `evaluatorCritique` tras un borrado de historial exitoso. **`setActiveStageId`** refresca `managerThreadId` vía `GET /ai-analysis/mdd/thread?stageId=` y **`fetchEstimation`**; **`createWorkshopStage`** (POST `/projects/:id/stages`) para multi-etapa. **`fetchWelcome`** envía `stageId`; **`persistMddContent`** y **`fetchEstimation`** envían `stageId` cuando hay etapa activa; **`sendMessage`** incluye `stageId` en el body del orquestador; **MDD stream** (manager, benchmark→`mdd/stream`, regenerate-section) incluyen `stageId` cuando aplica. Acciones: fetchProject, sendMessage (ai-orchestrator/chat), updateMddContent, persistMddContent (PATCH project + stageId opcional), **generateBenchmark** (POST /ai-analysis/stream, consume NDJSON, muestra progreso de agentes y persiste markdown en dbgaContent). Cuando la IA actualiza el MDD en la respuesta, set actualiza project/session/mddContent y el Semáforo y Costos se re-renderizan al instante. **Conformance vs MDD** se ejecuta automáticamente tras: (1) regenerar una sección del MDD (regenerate-section, comandos /); (2) generar o regenerar un entregable (generateBlueprint, generateApiContracts, generateLogicFlows, generateInfra). **`generateDeliverablesCascade`** → `POST /projects/:id/generate-deliverables` (cascada según `project.complexity`). |
