# Engine (SDD)

- **`mdd-markdown-parser.ts`:** `normalizeMddContent` / `parseMarkdownMdd` producen JSON compatible con `mddJsonSchema` (incluye **`constitution`**: señales «Constitución Cursor» desde §1–§2–§5 para el semáforo HIGH cuando Fase B fusione puertas con `sddDomainGraphOk`). Tests: `mdd-markdown-parser.constitution.spec.ts`.
- **`semaphore.service.ts`:** calcula `Stage.status` / `precisionScore` según `Project.complexity` (`ComplexityLevel`): **LOW** (HU + tareas), **MEDIUM** (spec o UC, API, UX o flujos, **historias de usuario**, tareas — 5 gates; 5/5 VERDE, ≥3 AMARILLO), **HIGH** (MDD JSON + Figma si `hasUxTeam`). En HIGH, si `sddDomainGraphOk` (Grafo Falkor sin dependencias huérfanas §3–§4), puede pasar a VERDE con `precisionScore` 92 aunque falten textos largos de `edge_cases`/`field_types`. Si `mddJson.constitution.template_detected` (Constitución Cursor), puertas extra (mapa, glosario, Gherkin, bloqueantes, ADR/stack): incumplimiento → AMARILLO; se fusiona con el resultado base (un VERDE puede bajar; un AMARILLO solo si la constitución es peor — score más bajo).
- **`mdd-update-pipeline.service.ts`:** valida/sanitiza MDD, sincroniza el borrador al Grafo SDD (`GraphMemoryService`) y evalúa `evaluateSddDependencyHealth` antes del semáforo cuando hay `projectId`/`stageId` y complejidad HIGH.
- **Costes / conformidad:** `cost-calculator.service.ts` delega la fórmula en `@theforge/business-rules`; `conformance.service.ts`; reparación post-generación Blueprint en `blueprint-conformance-repair.util.ts`; Contratos API en `api-conformance-repair.util.ts` (normalización `{id}`/`:id`, filas §4 faltantes).
- **Auditoría MDD determinística:** `mdd-internal-audit.util.ts` (paridad SQL/Mermaid, trazabilidad MFA) — compartido con el nodo Auditor y `EstimationService`.
- **Cabecera de documentos:** `document-date-header.util.ts` — `prependDocumentTimestamps` antepone en cada persistencia de markdown `Creado` y `Última regeneración` con fecha y hora completas (segundos, UTC) más comentario HTML `theforge-doc:created|updated`. Aplicado en entregables SDD (`stage-deliverable-persist`), MDD/BRD/DBGA (`projects.service`), change spec, pantallas MCP y legacy BRD.
- **Cascada SDD cross-artifact:** `sdd-cross-artifact.util.ts` (Spec↔MDD, HU↔UC, Tasks↔Blueprint/API); `phase0-brd-spec-bridge.util.ts` (Phase0→BRD→Spec en greenfield). Consumidos por `SddIntegrationService.analyzeArtifacts`.
- **Exactitud cascada ≥90 (PLAN-CASCADE-90-ACCURACY):** `domain-inventory.util.ts` (capacidades BRD, CrudMatrix, ProcessInventory, auth-skew); `cascade-accuracy.util.ts` (DocAccuracyScore / TaskAccuracyScore). Wired en analyze + delivery gate (con BRD) + hard gate export (`REQUIRE_DOC_ACCURACY_90=true`).
- **P0–P2 (0.14.0):** `domain-inventory-persist.util.ts` (`Stage.domainInventory`); `compose-section3-from-inventory.util.ts` (stubs §3); task auditor dominio; aliases C1/T1.
- **MDD SSOT (v1.6+):** `domain-inventory-conformance.util.ts` (entidades DBGA faltantes en §3, tablas plataforma sin BRD); `platform-table-justify.util.ts` (ancla MCP/memoria/chat en BRD §3, MDD §1, Spec; auto-comentarios `[platform:*]` pre-gate); `brd-decision-log.util.ts` (cierre decision log); `api-prefix-unify.util.ts` (`/api/` → `/api/v1` en contratos); `entity-api-trace.util.ts` (entidad → §3 → API); `dbga-benchmark-matrix.util.ts`; `sdd-external-contracts.util.ts` (WebSocket gateway, Banxico, Polygon). Wired en delivery gate, W4 cascada y `collectConformanceGaps`.
