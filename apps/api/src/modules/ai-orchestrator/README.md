# AI Orchestrator

- **`AiOrchestratorService`:** chat y streaming del Workshop; persiste MDD/DBGA/UX/Spec y demás entregables de proyecto; **BRD** y **Manual To-Be** de la etapa activa vía **`patchStage`** (`---FIN_BRD---` / `---FIN_TOBE---` en la respuesta del modelo). Enriquece flujo **LEGACY** con `askCodebase`. **`POST /ai-orchestrator/chat` y `chat/stream`:** body puede incluir `images` (`parseChatImageAttachments`); opcionales `brdContent`, `toBeManualContent` (sincronía con el panel antes del stream); mensaje puede ir vacío si hay imágenes; HITL complejidad usa `"(Imagen adjunta)"` cuando no hay texto; `SessionsService` recibe `userImages` para el LLM y el log. **`chat/stream`:** keepalive SSE cada 15s (`sse-stream.util.ts`) para evitar timeouts de proxy/QUIC en respuestas largas del LLM.
- **Inyección:** usa `PROJECTS_ORCHESTRATOR_PORT` (incluye `patchStage` para BRD/To-Be por etapa) e `THEFORGE_ORCHESTRATOR_PORT` (`projects-service.port.ts`, `theforge-service.port.ts`) en lugar de las clases concretas, para poder sustituir `ProjectsService` / `TheForgeService` por mocks en pruebas (`useExisting` en los módulos Nest).
- **Prueba mínima de contrato:** `ai-orchestrator.di.spec.ts` (tokens únicos).
