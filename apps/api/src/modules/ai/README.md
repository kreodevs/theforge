# Módulo AI — LLM, prompts y contexto

- **`config/llm-config.ts`** — Runtime **OpenRouter** (chat + embeddings vía la misma API), `llmMaxTokens()` (techo global, default 32K), **perfiles de salida** (`resolveLlmMaxTokensForPurpose`, `resolveLlmMaxTokensForWorkshopTab`: chat 8K, documento 32K, ux-guide 16K, langgraph 16K, auditor 8K), `resolveChatModelChain()` / `hasChatModelFallback()`, `getLlmProvidersSnapshot()`. `LLM_EMBEDDINGS_PROVIDER=none` apaga embeddings.
- **`config/llm-model-fallback.ts`** — `runWithModelFallback`, `isModelExhaustionError` (402, quota, modelo no disponible, 429 opcional, OpenRouter «provider returned error» en 400/502/503, contexto demasiado largo).
- **`adapters/openrouter.adapter.ts`** — `OpenRouterAdapter`: `chat.completions` y `embeddings` con SDK `openai` (base `OPENROUTER_BASE_URL`); cadena de modelos solo si hay `OPENROUTER_CHAT_MODEL_FALLBACK(S)`.
- **`ai.factory.ts`** — Devuelve siempre `OpenRouterAdapter`.
- **`ai.service.ts`** — `generateResponse` / `generateResponseStream` / **`generateAuditorResponse`** (runtime `auditorChatModel`, mismos adaptadores BYOK/OpenRouter). El system prompt del Workshop se construye en **`workshop-system-prompt.builder.ts`** (`buildWorkshopSystemPrompt`, variantes `sync` | `stream`; incluye **`appendUxGuideStitchPolicy`**: Google Stitch solo proyectos **NEW** y tab **ux-ui-guide**; **LEGACY** prohíbe Stitch). Opción **`welcomeBrief`** (`GenerateResponseOptions`): system mínimo + sin pegar DBGA/MDD/Spec/etc. en system (p. ej. `generateWelcome`: el contexto va en el mensaje de usuario).
- **`workshop-system-prompt.builder.ts`** — Builder unificado del system prompt Workshop (tabs, delimitadores, contexto de documentos, tech docs, multimodal). Tests: `workshop-system-prompt.builder.spec.ts`.
- **`intent-classifier.service.ts`** — Heurística rápida `explore` | `direct_edit` | `mixed` (sin LLM).
- **`intent-router.service.ts`** — Router híbrido: heurística de alta confianza primero (incl. `lastAssistantMessage` para confirmaciones); si el mensaje es ambiguo (`confidence` &lt; 0.85), una llamada LLM ligera (`WORKSHOP_INTENT_ROUTER_PROMPT`, JSON `{ action, confidence, reasoning }`) decide `chat_only` | `edit_document` | `confirm_then_edit`. Cache por turno; desactivar LLM: `INTENT_ROUTER_LLM=0`. `assistantOfferedDocumentEdit` en `intent-router.util.ts`.
- **`POST /ai/mermaid/regenerate`** — Regenera diagramas Mermaid rotos/truncados para el visor (`MERMAID_REGENERATE_PROMPT` + `repairMermaidBlockBody`).
- **`ux-guide-llm-context.ts`** — `uxGuideLlmOptions(project)`: `projectTypeForUxGuide` + recortes de Spec, casos de uso, historias, flujos, arquitectura, API, DBGA, fase 0 para enriquecer la guía y el prompt Stitch del **producto**. El auto-match de design reference usa el **resumen MDD** (`buildMddContextForUxGuide`), no el documento íntegro.
- **`utils/mdd-ux-guide-brief.util.ts`** — `buildMddContextForUxGuide(mdd)` (~8K máx.): extrae §1 (producto/dominio), stack UI de §2, nombres de entidades §3, flujos UX de §5 y señales explícitas (B2B/B2C, WCAG, colores). Se inyecta en system prompt del tab **ux-ui-guide** vía `currentMddContent`; el fast path `composeDesignSystemFromRef` sigue usando MDD completo solo para heurística local.
- **`utils/mdd-deliverable-context.util.ts`** — `buildMddContextFor*` (50k budget): checklist de cobertura + secciones priorizadas del MDD para **todos** los entregables SDD (HU, CU, Blueprint, API, Flujos, Arq., Tasks, Infra, Spec, agent-governance).
- **`utils/agent-governance.util.ts`** — Genera el scaffold de gobernanza IA que se exporta a cada proyecto. Emite en **todas las complejidades** la **regla `alwaysApply` `theforge-doc-sync.mdc`**, la **skill `theforge-doc-sync/SKILL.md`**, `THEFORGE-LINK.md` y `mcp.json.example`: sincronizan el SDD con el código vía MCP `report_documentation_gap` (drift greenfield/pre-producción), con disparador **fin de sesión / pre-commit** (revisar `git diff` vs SDD) además del inline. Reconcilian la **etapa activa**; no abrir una etapa por cada desvío. Subflujo espejo en `references/workflows.md` («Doc gap sync», desde MEDIUM).
