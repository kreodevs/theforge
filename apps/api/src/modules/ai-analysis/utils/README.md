# Utils (ai-analysis)

Utilidades compartidas entre nodos MDD y Benchmark para evitar código duplicado.

- **parse-json.ts** – `parseJsonOrThrow(text, schema)`: parsea texto que puede contener JSON con o sin markdown code fence; usado por nodos MDD (manager, clarifier, security, integration, auditor). El agente Benchmark (scout, etc.) puede migrar a este util en el futuro para homologar.
- **mdd-sanitize.ts** – `normalizeMddFormat(draft)`: estandariza títulos (## 2. Modelo de datos, ## 3. Contratos de API), elimina líneas huérfanas con solo un número, envuelve SQL en ```sql si viene en texto plano, formatea JSON en Contratos de API (```json + pretty-print), e inserta `---` antes de cada `##` para separación visual (`ensureHorizontalRuleBeforeH2`). Corrige sección 2: SQL no cerrado antes de ### Diagrama / ```mermaid y encabezado pegado "### Diagrama entidad-relaciónmermaid" (`fixSection2UnclosedSqlAndGluedMermaid`). En Seguridad/Integración convierte JSON con `"heading"` o `"section"` + `"details"` a markdown (`sanitizeSeguridadIntegracionRawJson`, `convertSectionDetailsJsonToMarkdown`). Además descontamina el formato "bullet list as JSON lines" (cuando cada línea es una viñeta con un fragmento de JSON: `- {`, `- "title": ...`, `- "content": [`), reconstruyendo el JSON y convirtiéndolo a markdown legible (`unbulletAndParseSectionJson`, `objectWithContentArrayToMarkdown`). `jsonSectionToMarkdown`, `sanitizeContextSection`, `replaceContextWhenOnlyMetadata`, `preserveContextSectionIfSubstantial`, `ensureContratosSection`. **`sanitizeMddAtPersist`** (SSOT al persistir/exportar MDD): despega encabezados pegados (`fixGluedSection6Heading`, `splitInlineBoldLabelRuns`), desenvuelve fences huérfanos (`stripOrphanFenceWrappingProse`), cierra fences abiertos (`closeUnclosedCodeFencesInDraft`, incl. `dockerfile`), degrada prosa promovida a `###` en §4/§6/§7 (`demoteProseHeadingsInSections`), repara JSON anidado y llaves desplazadas en §4 (`repairNestedJsonFencesInDraft`, `repairDisplacedJsonBracesInContratos`), colapsa reglas horizontales rotas (`collapseInlineHorizontalRules`) y quita UI/UX en MVP API-only (`stripUiUxSectionForApiOnlyMvp`). Usado por nodos MDD y servicio al emitir markdown. Tests: `mdd-sanitize.spec.ts`.
- **suggest-mdd-governance-patterns.util.ts** – Preselección de ids del wizard SSOT desde DBGA, resumen Fase 0 y BRD (LLM + heurística).
- **mdd-diagram-suggestions.ts** – `suggestMddDiagrams(draft)`: analiza el MDD y devuelve sugerencias de diagramas Mermaid a insertar (erDiagram en modelo de datos si hay CREATE TABLE; stateDiagram-v2 en contratos si hay auth/login; flowchart en frontend si hay componentes). **`sqlToErDiagramContent(sql)`** deriva el erDiagram de forma determinista desde `CREATE TABLE`/`REFERENCES` (SSOT; pisa salida del LLM). `injectMddDiagrams(draft, suggestions)` inserta o **reemplaza** el bloque ER existente. `regenerateErDiagramFromSql(draft)`: regenera solo el erDiagram de la sección 3 desde el SQL. Usado por el nodo `mdd-diagram-injector`, `prepareMddForOutput`, `parseModeloDatosFromSection3Markdown` y por el comando del Manager "regenerar diagrama ER".
- **mdd-pre-render.ts** – Re-export desde `engine/mdd-pre-render.ts`. `preRenderMddSanity(draft)`: valida Mermaid (sin coma PK/FK, espacios normalizados) y tablas markdown en §4 (alineación segunda fila, sin líneas en blanco). Devuelve `{ ok, code?, message? }`; códigos `ERR_MERMAID_SYNTAX`, `ERR_TABLE_SYNTAX`. Usado en `projects.service` antes de persistir `mddContent`.
- **mdd-complexity-rigor.ts** – Apéndices de prompt según `mddComplexity` (LOW/MEDIUM/HIGH): `clarifierComplexityAppendix`, `softwareArchitectComplexityAppendix`, `auditorConstitutionRigorAppendix`, `contextSynthesizerComplexityAppendix` (regenerar §1). Alineado con la relajación de gaps en `EstimationService` para el mismo nivel.
- **mdd-inject-ui-mcp-frontend.util.ts** – Si hay MCP gráfico compatible activo, inyecta su librería en **§2 Frontend → Stack UI → UI Library** (`injectUiMcpIntoMddFrontendSection`, append ` + {label}` idempotente). `formatUiMcpLibraryLabel` deriva el texto desde `getActiveCompatibleMeta`. `buildUiMcpFrontendArchitectHint` alimenta el prompt del Arquitecto. Integrado en `prepareMddForOutput`, `mdd-prepare-output.node` y `createMddGraph*`. Tests: `mdd-inject-ui-mcp-frontend.util.spec.ts`.
