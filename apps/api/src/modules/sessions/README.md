# Sessions

Gestión de sesiones de chat del Workshop y parseo de respuestas que incluyen documentos (MDD, UX/UI, DBGA, BRD por etapa, Manual To-Be, etc.).

- **SessionsService:** CRUD de sesiones, `chat`, `chatStream`, `generateWelcome`. Extracciones Fase 3: **`session-chat-llm-options.util.ts`** (opciones LLM), **`session-chat-response-parse.util.ts`** (delimitadores + fallbacks sync/stream), **`session-chat-turn.runner.ts`** (`processSessionChatTurnOutcome`, `buildSessionChatDonePayload`). `chat` / `chatStream` delegan el post-procesamiento al runner; lo consume **`AiOrchestratorService`** (no es el contrato HTTP de `POST :id/chat`). **`generateWelcome`:** `welcomeBrief: true` + contexto acotado en user prompt. **Intención:** `IntentRouterService` + `applyIntentPersistGate` en el runner. **Imágenes:** visión vía `describeImagesForChat`; stream usa solo texto al LLM (`sessionHistoryToLlm`). **DBGA:** `tryBenchmarkDbgaEditTurn` en fast path benchmark.
- **`POST :id/chat` (SessionsController):** delega en **`AiOrchestratorService.chatBySessionId`** — misma lógica que **`POST /ai-orchestrator/chat`** (HITL complejidad, PATCH MDD/UX/DBGA desde body, supervisor, legacy/TheForge, **`uxGuideLlmOptions`** cuando toca, ingest SDD, evaluador). Body: `message` y/o **`images`** (`parseChatImageAttachments`), opcionales `activeTab`, `stageId`, `mddContent`, `uxUiGuideContent`, `dbgaContent`. Respuesta: **`{ session, project, uxUiGuideContent?, evaluatorCritique? }`** (no el retorno plano interno de `SessionsService.chat`).
- **ChatResponseParserService:** Separa documento vs mensaje de chat (delimitadores `---FIN_MDD---`, `---FIN_DBGA---`, etc.), fusiona secciones de MDD/DBGA y fallback cuando el modelo omite el delimitador (incl. Fase 0 / tab `benchmark`, títulos tipo *Research Report*, outline numerado, intro «He integrado…» + cuerpo). **`looksLikeApiEndpointCatalog` / `mergeApiEndpointCatalogIntoDbga`:** listas GET/POST se fusionan en §11 (no sustituyen ni duplican el H1). Respuestas del modelo que reinician un DBGA truncado se deduplican en lugar de concatenar tras `---`. `SessionsService.salvageDbgaFromAssistantText` recupera DBGA pegado solo en el chat. Si el modelo responde solo «He actualizado…» sin `---FIN_DBGA---`, `refineDbgaFromUserRequest` relanza el refinado con `BENCHMARK_REFINE` y el DBGA actual. Con **imágenes**, la descripción de visión va al refinado (no solo el texto corto del usuario). `benchmarkAssistantChatMessage` colapsa el cuerpo del documento en el chat cuando sí se persistió (`looksLikeDbgaDocumentBody`). Tab **benchmark** + sub-tab Fase 0 edita `dbgaContent`; el Benchmark (Deep Research) es `phase0SummaryContent` (no se pisan entre sí).
- **document-content.util.ts:** Funciones puras `normalizeDashes`, `stripChatLabel`, `cleanDocumentContent` (formatea markdown y garantiza `## Registro de cambios del documento` con fila 1.0 si falta). Sin DI.
- **document-shrink.util.ts:** `wouldShrinkDocDangerously`, `validateDocumentForPersist`, `documentPersistFieldLabel` — evita persistir documentos vacíos, solo-changelog o que borren la mayor parte del contenido previo (chat orquestador, `clarify-spec`, `PATCH specContent`, `PATCH dbgaContent`, `legacy/generate-from-codebase`).
- **`POST project/:projectId/salvage-dbga`:** recupera DBGA del chat benchmark; snapshot del estado actual antes de restaurar. Ver también **`GET/POST …/document-snapshots`** en Projects.
- **orchestrator-doc-guard.util.ts:** Heurísticas compartidas para tabs del orquestador (Arquitectura, Spec, Infra, etc.): si el usuario pidió cambio o el asistente afirma haber editado pero no hubo persistencia, `SessionsService` añade aviso en el chat y el frontend muestra banner — **sin pedir al usuario delimitadores `---FIN_*---`** (mensajes en `@theforge/shared-types/workshop-fin-delimiter-covenant`).
