# Sessions

Gestión de sesiones de chat del Workshop y parseo de respuestas que incluyen documentos (MDD, UX/UI, DBGA, BRD por etapa, Manual To-Be, etc.).

- **SessionsService:** CRUD de sesiones, `chat`, `chatStream`, `generateWelcome`. Pipeline unificado: **`prepareSessionChatTurn`** → LLM sync/stream → **`completeSessionChatTurn`**. Utils: **`session-chat-turn.runner.ts`** (stream buffer, log entries, outcome), **`session-chat-llm-options.util.ts`**, **`session-chat-response-parse.util.ts`**. Consumido por **`AiOrchestratorService`**. **Intent:** `IntentRouterService` (heurística + LLM); pedidos DBGA largos no cortocircuitan solo por keywords (confianza ~0.78 → LLM). **Refinado DBGA:** prefiere `---FIN_DBGA---`; si el modelo (p. ej. Gemma) omite el marcador pero devuelve un DBGA completo (título + tamaño), `acceptCompleteDbgaWithoutFinMarker` lo acepta. `isDbgaEditEffectivelyUnchanged` no descarta renombres cortos (PAT Wasender).
- **`POST :id/chat` (SessionsController):** delega en **`AiOrchestratorService.chatBySessionId`** — misma lógica que **`POST /ai-orchestrator/chat`** (HITL complejidad, PATCH MDD/UX/DBGA desde body, supervisor, legacy/TheForge, **`uxGuideLlmOptions`** cuando toca, ingest SDD, evaluador). Body: `message` y/o **`images`** (`parseChatImageAttachments`), opcionales `activeTab`, `stageId`, `mddContent`, `uxUiGuideContent`, `dbgaContent`. Respuesta: **`{ session, project, uxUiGuideContent?, evaluatorCritique? }`** (no el retorno plano interno de `SessionsService.chat`).
- **ChatResponseParserService:** Separa documento vs mensaje de chat (delimitadores `---FIN_MDD---`, `---FIN_DBGA---`, etc.), fusiona secciones de MDD/DBGA y fallback cuando el modelo omite el delimitador (incl. Fase 0 / tab `benchmark`, títulos tipo *Research Report*, outline numerado, intro «He integrado…» + cuerpo). **`looksLikeApiEndpointCatalog` / `mergeApiEndpointCatalogIntoDbga`:** listas GET/POST se fusionan en §11 (no sustituyen ni duplican el H1). Respuestas del modelo que reinician un DBGA truncado se deduplican en lugar de concatenar tras `---`. `SessionsService.salvageDbgaFromAssistantText` recupera DBGA pegado solo en el chat. Si el modelo responde solo «He actualizado…» sin `---FIN_DBGA---`, `refineDbgaFromUserRequest` relanza el refinado con `BENCHMARK_REFINE` y el DBGA actual. Con **imágenes**, la descripción de visión va al refinado (no solo el texto corto del usuario). `benchmarkAssistantChatMessage` colapsa el cuerpo del documento en el chat cuando sí hubo persistencia. **`finalizeBenchmarkTurn`:** si el asistente pide confirmación (`isAssistantAwaitingDocumentEditApproval`), no persiste aunque venga `---FIN_DBGA---`; puede anteponer la propuesta al mensaje de chat. **`shouldWarnOrchestratorDocNotPersisted`:** no avisa en flujos de aprobación pendiente ni cuando el modelo emitió delimitador sin pedido de edición del usuario. **`parseBenchmarkResponse` (dbga-edit.util):** el marcador `---FIN_DBGA---` se detecta con `FIN_DBGA_MARKER_RE`, tolerante a variantes reales del modelo (sin guiones finales `---FIN_DBGA`, sin iniciales `FIN_DBGA---`, en su propia línea o envuelto `## FIN_DBGA` / `**FIN_DBGA**`). **`isDbgaContentNearlyIdentical`:** tolerancia de longitud **acotada** (cap 200 chars) para que ediciones pequeñas en DBGAs grandes (~90 KB) no se descarten como «sin cambios». Tab **benchmark** + sub-tab Fase 0 edita `dbgaContent`; el Benchmark (Deep Research) es `phase0SummaryContent` (no se pisan entre sí).
- **document-content.util.ts:** Funciones puras `normalizeDashes`, `stripChatLabel`, `cleanDocumentContent` (formatea markdown y garantiza `## Registro de cambios del documento` con fila 1.0 si falta). Sin DI.
- **document-shrink.util.ts:** `wouldShrinkDocDangerously`, `validateDocumentForPersist`, `documentPersistFieldLabel` — evita persistir documentos vacíos, solo-changelog o que borren la mayor parte del contenido previo (chat orquestador, `clarify-spec`, `PATCH specContent`, `PATCH dbgaContent`, `legacy/generate-from-codebase`).
- **`POST project/:projectId/salvage-dbga`:** recupera DBGA del chat benchmark; snapshot del estado actual antes de restaurar. Ver también **`GET/POST …/document-snapshots`** en Projects.
- **orchestrator-doc-guard.util.ts:** Heurísticas compartidas para tabs del orquestador (Arquitectura, Spec, Infra, etc.): si el usuario pidió cambio o el asistente afirma haber editado pero no hubo persistencia, `SessionsService` añade aviso en el chat y el frontend muestra banner — **sin pedir al usuario delimitadores `---FIN_*---`** (mensajes en `@theforge/shared-types/workshop-fin-delimiter-covenant`).
