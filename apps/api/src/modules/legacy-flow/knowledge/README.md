# Knowledge pack – Flujo legacy

Contenido estático derivado de los tres cuadernos de NotebookLM usados como base de conocimiento del flujo legacy:

1. **Arquitectura de Prompts y Patrones** → `arquitectura-prompts-patrones.md`
2. **Specification-Driven Development and the Evolution of AI Engineering** → `specification-driven-development.md` (incluye estructura canónica del MDD: 7 secciones y cascada Constitution → Spec → Plan → Tasks).
3. **Architecting Agentic Systems: Frameworks, Patterns, and Advanced Workflows** → `architecting-agentic-systems.md` (contexto general del coordinador; el entregable **Arquitectura** usa `architecture-prompt.md`: arquitectura del **producto**, no diseño multi-agente salvo evidencia en repo).

Los servicios LegacyCoordinator y LegacyReviewer cargan este contenido en sus system prompts. Para actualizar el conocimiento, reemplaza los `.md` por exports reales de los cuadernos o por un paquete curado.
